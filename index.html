What is the Antrea Operator for Kubernetes

The Antrea Operator installs and manages Antrea components in Kubernetes clusters.
The Operator is designed to be installed in the early stages of cluster deployment so that it will install Antrea as the default container networking solution for every node in the cluster.
The Antrea operator will take care of deploying, managing and monitoring:

  • The antrea-controller deployment
  • The antrea-agent daemonset

The Antrea Operator support deployment as a cluster network operator for Openshift 4.
It monitors the cluster's network CRD to update container and service network CIDRs used by Antrea.
The Antrea Operator also monitors a dedicated CRD (AntreaInstall), which defines parameter specific to Antrea configuration that are then applied to the Antrea Controller and Agent.

For more information about the Antrea operator, see the >README file

For more information about Kubernetes networking with Antrea, please visit the Antrea project homepage
